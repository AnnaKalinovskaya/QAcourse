3.	Поддержка темной темы. 
4.	Строку состояния не следует кастомезировать.
5.	Если пользователь не желает залогиниться через аккаунт Apple, следует предоставить функцию автозаполнения для пароля. 
6.	Не следует сразу после запуска приложения отсылать пользователя на страницу с настройками. 
7.	Настраивать уведомления пользователь может непосредственно в самом приложении.
8.	Для уведомления пользователя о способе его идентификации вместо иконок используйте кнопки с текстом (например "Sign In with Face ID")
9.	Каждому действию с уведомлением соответствует простой узнаваемый глиф.
10.	Уведомления с предупреждением имеют 2 кнопки.
11.	Следует избегать расположения интерактивных элементов в самом низу экрана и в его углах.
12.	Не следует использовать в приложении термин «passcode».
13.	Поддержка полноэкранного и оконного режима при просмотре видео.
14.	Картинка при просмотре по умолчанию не интерактивна.
15.	Функция печати должна быть легкодоступна.
16.	Не следует создавать кнопки на всю ширину экрана.
17.	Кнопка для отмены действия должна называться «Cancel».
18.	При текстовом поле должна быть кнопка для очистки введенных данных. 
19.	Popover не должен закрывать важную информацию или элемент при нажатии на который, popover появился. 
20.	Следует избегать предлагать пользователю совершить деструктивные действия.
21.	Углы иконки приложения острые, фон на иконке простой.
